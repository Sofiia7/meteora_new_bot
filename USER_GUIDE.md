# Инструкция по использованию — Meteora LP-Farm Bot

Как бот отбирает токены, что настраивается, как входит в позицию, сколько раз и какие пулы
присылает, и как следит за открытой позицией. Все значения — из фактического кода
(`src/shared/config.ts` + сервисы), не из спеки, поэтому отражают то, что реально работает.

---

## 1. Откуда берутся токены-кандидаты (Scanner)

Работает в фоне каждые **`SCANNER_INTERVAL_MS`** (по умолчанию 300000 = 5 минут). Источники —
независимые, ошибка одного не блокирует другой:

- **DexScreener token-profiles** — только СВЕЖЕподанные профили (узкое окно, до 30 адресов за тик).
- **Meteora DLMM pools API** (`dlmm.datapi.meteora.ag`, топ-50 по объёму за 24ч) — ловит активные
  пулы независимо от возраста токена (например старый, но резко объёмный токен).

### Жёсткие фильтры (после них — либо кандидат, либо тихо отсеян, без уведомления)

| Параметр | .env | По умолчанию | Смысл |
|---|---|---|---|
| Min Market Cap | `MIN_MARKET_CAP` | 250 000 $ | ниже — отсекается |
| Min Volume 24h | `MIN_VOLUME_24H` | 1 000 000 $ | ниже — отсекается |
| Max возраст токена | `MAX_TOKEN_AGE_DAYS` | 21 день | старше — отсекается (если возраст вообще известен источнику) |
| Сеть | — (жёстко) | только Solana | не Solana — отсекается |

**Дедупликация:** один и тот же адрес токена повторно не уведомляется в течение **24 часов**
(захардкожено, не настраивается через `.env`), кроме отдельного механизма ATH re-notify (см. §6).

---

## 2. Chart-Health score 0–100 (после фильтров сканера)

Реальные 15-минутные свечи берутся с **GeckoTerminal**. Итоговый скор стартует со 100 и штрафуется:

| Фактор | Штраф | Настройка |
|---|---|---|
| Просадка от ATH > `MAX_ATH_DISTANCE_PCT` (30% по умолч.) | до −60, пропорционально глубине просадки | `MAX_ATH_DISTANCE_PCT` |
| Объём 24h < цели (`MIN_VOLUME_24H`) | до −20 (по тиру: ≥3x цели=100 очков объёма … <0.5x=10 очков) | цель = `MIN_VOLUME_24H` |
| Ликвидность < 2% от Market Cap | −10 | захардкожено |
| Дамп 24h < −50% | −25 | захардкожено |
| Просадка 24h < −25% | −10 | захардкожено |
| RSI(15м) > 75 (перегрет) | −8 | захардкожено |

**Проходит дальше, если:** `score ≥ MIN_HEALTH_SCORE` (по умолчанию **65**) **и** просадка от ATH < 90%.

---

## 3. Security score 0–100 (после chart-health)

Три независимых источника опрашиваются параллельно, скор стартует со 100:

| Источник | Что проверяет | Штраф |
|---|---|---|
| **GeckoTerminal** | Twitter/соц-активность, `gt_score` (доверие 0–100) | нет твиттера −5; gt_score<30 → −20; 30–50 → −8 |
| **RugCheck** | риск-скор `score_normalised` (0=безопасно…100=риск), danger-риски | риск≥60 → −30; риск 30–60 → −12; danger-флаг → −25 |
| **BubbleMaps** | `decentralisation_score` (концентрация держателей) | <15 → −25; 15–30 → −12 |

**Fail-closed:** если источник недоступен (сеть упала / API лимит) — это штраф, а не "всё ок":
GeckoTerminal недоступен считается отдельно, RugCheck недоступен −30, BubbleMaps недоступен −15.

**Hard-fail (мгновенный отказ, скор не спасает):**
- 🍯 honeypot (по данным RugCheck)
- активная mint authority (можно доминтить токен)
- активная freeze authority (можно заморозить кошельки)

**Проходит дальше, если:** нет hard-fail **и** `score ≥ MIN_SECURITY_SCORE` (по умолчанию **60**).
Параметр `MAX_HOLDER_CONCENTRATION_PCT` (50%) в `.env` есть, но в текущей формуле скоринга не
используется напрямую — децентрализация учитывается через BubbleMaps выше.

> GMGN отдаёт 403 с VPS (блокировка Cloudflare) — параметр `MIN_GMGN_FEES_SOL` в `.env` больше не
> используется, security полностью на GeckoTerminal + RugCheck + BubbleMaps.

## 3.1. AI второе мнение (опционально, выключено по умолчанию)

`AI_ENABLED=true` подключает **локальную** LLM (Ollama/LM Studio/vLLM, свой сервер, без
облака) — она получает метрики + результат security и добавляет короткий вердикт с уровнем риска
🟢/🟡/🔴 прямо в уведомление. Ни на что не влияет — не блокирует и не пропускает автоматически,
только доп. информация для тебя.

## 4. Telegram-уведомление о токене

Присылается один раз на прошедший фильтры токен: метрики (mcap, объём, security-скор,
chart-health), причины штрафов (если есть), и **7 ссылок**: GMGN, Axiom, BubbleMaps, RugCheck,
DexScreener, Solscan, Photon. После этого бот переходит к поиску пулов Meteora (§5) — вход
предлагается уже там.

Если security не прошёл — есть отдельная кнопка "force enter" (войти вопреки провалу), решение
всё равно за тобой.

---

## 5. Поиск пула для входа (Pool Watcher)

- Опрашивает **каждые `POOL_CHECK_INTERVAL_MS`** (по умолчанию 30 сек).
- Источник: DexScreener `token-pairs` (все пары с `dexId=meteora` — и DLMM, и DAMM V2). Для DLMM
  дополнительно дообогащается реальным fee/bin_step с `dlmm.datapi.meteora.ag` (DexScreener этого
  не отдаёт).
- **Показывает ВСЕ найденные пулы Meteora по токену, без фильтрации** — решение сделано осознанно:
  бот никогда не выбирает пул сам. Сортировка по TVL (сначала самый ликвидный).
- **Сколько кнопок:** до `POOL_BUTTONS_MAX` (по умолчанию **8**) пулов получают кнопку "✅ Войти N"
  с типом пула, fee% и TVL. Пул, совпадающий с целевой стратегией (fee **5%** + bin step
  **80, 100 или 125** — эти три значения захардкожены, не в `.env`), помечается **⭐**; это только
  метка, войти можно в любой из показанных.
- **Сколько раз шлёт:** первый раз — как только нашёлся хоть один пул. Повторно — **только** если
  появился НОВЫЙ пул с TVL ≥ `POOL_RENOTIFY_MIN_TVL` (по умолчанию **1000 $**); мелкие
  свежесозданные DAMM-пулы ($0–14 TVL) не триггерят повторную рассылку всего списка.
- **Если пулов пока нет вообще** — одно уведомление "наблюдаю" и продолжается опрос.
- **Таймаут наблюдения:** `POOL_WATCH_TIMEOUT_MS` (по умолчанию **2 часа**). По истечении, если
  пулов так и не было (или были, но не вошли) — предложение продолжить наблюдение или токен уходит
  в `/watchlist` (ручное управление: `/watchlist`, `/watch <CA>` для добавления).

## 6. Повторное уведомление на новом ATH (ATH-watcher)

Раз в 2 минуты (захардкожено) проверяет уже известные (прошедшие security) токены **без открытой
позиции**. Если текущая цена превышает последний уведомлённый ATH минимум на `ATH_RENOTIFY_PCT`
(по умолчанию **10%**) и прошёл `ATH_RENOTIFY_COOLDOWN_MIN` (по умолчанию **30 минут**) с прошлого
уведомления — **полностью перезапускает** chart-health + security (ловит отложенный раг) и, если
всё ещё проходит фильтры, шлёт уведомление заново, с теми же кнопками, как при первом обнаружении.

---

## 7. Вход в позицию (LP Manager)

- Только по кнопке — бот никогда не входит сам.
- **Single-sided SOL**: депонируется только SOL, без отдельного свопа в мемкоин. Диапазон бинов —
  строго по ОДНУ сторону от активного бина (это требование самого DLMM SDK: диапазон целиком выше
  активного бина принимает только не-SOL токен, целиком ниже — только SOL, если пытаться дать оба —
  реально задепонируется 0). Направление зависит от того, какой токен пула — X или Y.
- Ширина диапазона — `PRICE_RANGE_UPPER` (по умолчанию **100%** от текущей цены), стратегия
  распределения внутри диапазона — **BidAsk** (концентрация у дальнего края, а не равномерно).
- **Размер позиции:** `LP_AMOUNT_SOL` (по умолчанию 0.1 SOL — на текущем сервере выставлено 0.02).
- **Лимит открытых позиций одновременно:** `MAX_POSITIONS` (по умолчанию **3**).
- **Проверка баланса перед входом:** нужно ≥ `LP_AMOUNT_SOL` + rent (~0.07 SOL) +
  `max(WALLET_SAFETY_BUFFER_SOL, LP_AMOUNT_SOL × WALLET_SAFETY_BUFFER_PCT%)`. Не хватает — вход
  отменяется, шлётся уведомление о причине.
- **Проскальзывание на выходном свопе:** `SWAP_SLIPPAGE_BPS` (по умолчанию 100 = 1%).

## 8. Мониторинг открытой позиции и выход

Два независимых монитора работают параллельно, оба вызывают один и тот же код закрытия позиции
(claim fees → remove liquidity → своп остатка мемкоина в SOL → PnL по дельте баланса кошелька):

### Exit Strategy (тик раз в 60 сек) — тейк-профиты, авто-закрытие
| Сигнал | Условие | Настройка |
|---|---|---|
| Цель по комиссиям | накопленные unclaimed fees ≥ `EXIT_FEE_THRESHOLD` от размера позиции | по умолч. 5% |
| Новый ATH | цена > 105% от последнего сигнального ATH (после ≥5 точек истории) | захардкожено +5% |
| Пробой Bollinger Bands | цена выше верхней полосы (период `BOLLINGER_PERIOD`=20, `BOLLINGER_STD_DEV`=2) | |
| Деградация графика (RSI<`RSI_DEGRADATION_THRESHOLD`=40 + низкий объём) | **только предупреждение** с кнопкой, авто-закрытия НЕТ | |
| Ценовой стоп-лосс | **выключен по умолчанию** (`ENABLE_PRICE_STOP_LOSS=false`) — осознанное решение против ложных срабатываний на волатильности мемов | |

### Panic Detector (тик раз в 30 сек) — композитная защита от рага
Авто-закрытие только когда **≥ `PANIC_REQUIRED_FACTORS`** (по умолчанию **2**) из пяти факторов
активны ОДНОВРЕМЕННО в окне `PANIC_TIME_WINDOW_MIN` (по умолчанию **15 минут**):

| Фактор | Условие | Настройка |
|---|---|---|
| F1 volume_drop | 24h-объём ниже (1 − `PANIC_VOLUME_DROP_PCT`%) от порога сканера | 60% |
| F2 rsi_low | RSI(14) по истории цен ниже `PANIC_RSI_THRESHOLD` | 40 |
| F3 price_from_ath | просадка от ATH-в-позиции ≥ `PANIC_PRICE_DROP_FROM_ATH_PCT` | 50% |
| F4 security_degraded | повторный security-чек нашёл hard-fail (отложенный раг); троттлится `SECURITY_RECHECK_MIN` | 5 мин |
| F5 tvl_drop | TVL пула упал ≥ `PANIC_TVL_DROP_PCT` от пика; троттлится так же | 50% |

### Ручной выход
Кнопка 🔴 прямо на сообщении об открытой позиции — работает в любой момент, это главный
инструмент паники оператора.

## 9. Telegram-команды

| Команда | Что делает |
|---|---|
| `/start` | запуск/приветствие |
| `/status` | текущий статус бота |
| `/positions` | список открытых позиций с кнопкой закрытия |
| `/watchlist` | список отслеживаемых токенов (+ удаление) |
| `/watch <CA>` | добавить токен в наблюдение вручную |
| `/scan <CA>` | проверить конкретный токен вручную (в обход сканера) |

Управлять ботом (жать кнопки, спускать деньги) может только чат из `TELEGRAM_ALLOWED_CHAT_IDS`
(если пусто — только `TELEGRAM_CHAT_ID`); остальные апдейты режутся мидлварой до хендлера.

## 10. Полная таблица настраиваемых параметров (`.env`)

```
# Безопасность
DRY_RUN=true                       # true = ничего не торгует, только логирует
ENABLE_MAINNET_TRADING=false       # реальные сделки только при DRY_RUN=false И этом true

# Сканер
SCANNER_INTERVAL_MS=300000         # 5 мин — период сканирования новых токенов
MIN_MARKET_CAP=250000              # $ минимальный market cap
MIN_VOLUME_24H=1000000             # $ минимальный объём 24ч
MAX_TOKEN_AGE_DAYS=21               # старше — не берём

# Chart-health
MIN_HEALTH_SCORE=65                # порог прохождения скора графика
MAX_ATH_DISTANCE_PCT=30            # % просадки от ATH, дальше — штраф

# Security
MIN_SECURITY_SCORE=60              # порог прохождения security-скора
MAX_HOLDER_CONCENTRATION_PCT=50    # есть в конфиге, в скоринге пока не используется напрямую
SECURITY_RECHECK_MIN=5             # троттлинг периодического re-check для активных позиций

# Pool watcher
POOL_CHECK_INTERVAL_MS=30000       # 30 сек — как часто ищем пулы
POOL_WATCH_TIMEOUT_MS=7200000      # 2 часа — таймаут наблюдения
POOL_BUTTONS_MAX=8                 # сколько пулов кнопками (остальное — без кнопок)
POOL_RENOTIFY_MIN_TVL=1000         # $ TVL нового пула, чтобы триггернуть повторную рассылку

# LP
LP_AMOUNT_SOL=0.1                  # размер каждой позиции в SOL
MAX_POSITIONS=3                    # сколько позиций одновременно
PRICE_RANGE_UPPER=100              # % ширины диапазона входа от текущей цены
SWAP_SLIPPAGE_BPS=100              # 1% — проскальзывание на выходном свопе
WALLET_SAFETY_BUFFER_SOL=0.1       # минимальный буфер SOL сверх позиции+rent
WALLET_SAFETY_BUFFER_PCT=15        # или % от размера позиции — берётся максимум из двух

# Exit strategy
EXIT_FEE_THRESHOLD=0.05            # 5% — цель по комиссиям для авто-выхода
BOLLINGER_PERIOD=20
BOLLINGER_STD_DEV=2
RSI_DEGRADATION_THRESHOLD=40       # ниже — предупреждение (не авто-выход)
VOLUME_DEGRADATION_RATIO=0.3
ENABLE_PRICE_STOP_LOSS=false       # жёсткий SL выключен по умолчанию (продуктовое решение)
EXIT_STOP_LOSS_PERCENT=35          # используется только если включить строку выше

# Panic detector (композитный)
PANIC_REQUIRED_FACTORS=2           # сколько факторов нужно одновременно
PANIC_TIME_WINDOW_MIN=15
PANIC_VOLUME_DROP_PCT=60
PANIC_RSI_THRESHOLD=40
PANIC_PRICE_DROP_FROM_ATH_PCT=50
PANIC_TVL_DROP_PCT=50

# ATH re-notify
ATH_RENOTIFY_PCT=10
ATH_RENOTIFY_COOLDOWN_MIN=30

# AI (опционально)
AI_ENABLED=false
AI_BASE_URL=http://localhost:11434/v1
AI_MODEL=llama3.1
```

### Что НЕ настраивается через `.env` (зашито в коде)
- Целевая стратегия для метки ⭐ у пула: fee 5% (500 bps) + bin step 80/100/125.
- Дедупликация токенов сканером: 24 часа.
- Список "мажорных" минтов (SOL/USDC/USDT), которые не считаются токеном-кандидатом.
- Все веса штрафов в security-скоринге (honeypot/mint/freeze — hard-fail; конкретные проценты
  штрафов по каждому источнику).
- Веса штрафов в chart-health (за исключением двух порогов выше).
- Периоды опроса ATH-watcher (2 мин) и Exit Strategy (60 сек) / Panic Detector (30 сек).
